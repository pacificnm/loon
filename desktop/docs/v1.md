# Loon Admin Desktop v1 — Plan

## Status: Scaffolded

Desktop admin GUI for library management, TMDB matching, scans, and server settings. Complements the webOS client admin screen with a keyboard/mouse-friendly desktop experience.

## Goals

- Manage Loon server from a native desktop window (scan, match, library paths, logs)
- Reuse Nest `nest-gui` host and `nest-theme` lifecycle
- Loon-branded dark theme aligned with webOS client (`#090B10` / `#7DD3FC`)
- Talk to existing Loon HTTP API (`loon-server`) — no duplicate business logic

## Non-goals (v1)

- Replace webOS client
- Embed server in the GUI process (client/server stay separate)
- Full design system / component library (`nest-ui` deferred)
- Docking, command palette, in-app log viewer

## Crate boundaries

| Crate | Depends on | Owns |
|-------|------------|------|
| `loon-admin` | `nest-gui`, `nest-theme`, `loon-egui-theme` | Views, navigation, HTTP client to server |
| `loon-egui-theme` | `nest-theme`, `nest-design`, `egui` | Token → `egui::Visuals` mapping |
| `loon-server` | (existing) | API, scan, SQLite, streaming |
